GitHub Publishing Help Page

Before we begin, make sure to note that all commands must be typed exactly as shown.

  • First, open Git Bash by rigth clicking inside of the folder and selecting Git Bash. You should get the following screen.
  • Next use the command for initilizing your code. This is: git init
  • Then you must tell the program where to look for the information. This is done by using the command: git checkout -b gh-pages.
  • Now is a good time to check the status of your files. This can be done by simply typing: git status
    • It is important to note a few things here:
      1. If you get the following screen, then all of your files are properly saved.
      2. If there are files that show up in red however, then these files will need to be commited. We will review this process in a moment
  • Our next process is that of commiting.
    1. First, use the command: git add filename.
    2. Next, check the status of the file. It should be green now after using the command git status again.
    3. Now to commit these changes, use the command: git commit-m "" and in the quotaions, write a brief statement about what you did for your own reference later.
    4. Check your git status one mroe time and there should only be a short message in white text.
